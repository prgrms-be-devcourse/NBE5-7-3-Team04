### 원인

JPQL에서 여러 컬럼/엔티티를 SELECT하면 항상 Object[] 배열에 각각의 값을 담아서 반환하는데 내부적으로 쿼리 결과가 여러 개일 때 List<Object[]>를 반환한다. 만약 **단일 결과만 있다고 해도 실제로는 List<Object[]>가 반환될 수 있다**. 

처음에는 하나의 RefundId로 조회했을 때 하나의 결과만 나온다고 생각하고 Object[]로 받을 것을 예상했으나 사실 List<Object[]>가 반환되고 있었다.

### 해결결과

Repository : Object[] → List<Object[]>를 반환하는 것으로 변경

```java
@Query(""" 
        SELECT rf, res.quantity, sch.startTime, p
        FROM Refund rf
        JOIN Reservation res ON rf.reservationId = res.id
        JOIN PerformanceSchedule sch ON res.scheduleId = sch.id
        JOIN Performance p ON sch.performanceId = p.id
        WHERE rf.id = :refundId
        """)
List<Object[]> findRefundsDetailByRefundId(@Param("refundId") Long refundId);
```

Service : Repo 메서드 호출 후 results.get(0)으로 가장 첫번째 Refund만 매퍼에 넘겨줌으로써 mapper에는 Object[]만 들어갈 수 있도록 함

```java
/// refund id의 환불내역 디테일 조회. 없는 refundId인 경우 return null
public RefundDetailResponse findRefundsDetailByRefundId(Long refundId) {
    List<Object[]> results = refundRepository.findRefundsDetailByRefundId(refundId);
    
    // refundId로 조회한 결과가 없는 경우 get(0) 실행하기 전에 검사
    if (results.isEmpty()) {
        throw ErrorCode.REFUND_NOT_FOUND.domainException("존재하지 않는 환불입니다. refundId: " + refundId);
    }
    return refundDetailMapper.toRefundDetailResponse(results.get(0));
}
```
