## 모듈의 역할 통일

-   **Entity**
    -   (JPA Entity 기능을 같이 사용하면서) 데이터 보관
    -   유효성 검사 (cancel 같은거)
    -   간단한 필드변경
    -   Required 필드는 원시 타입으로 지정
    -   Nullable은 래퍼 타입으로 지정
-   **Service**
    -   모든 비즈니스 흐름 조합 (예약, 공연관리, 취소, 환불) 익숙한 스타일
-   **Repository**
    -   CRUD만
-   **Controller**
    -   Request → Service 호출만
-   **DTO**
    -   요청/응답 전용 (Entity 반환 X)
    -   Record 사용하기

## 커밋 메시지

[https://velog.io/@devshk447/나만-몰랐던-Conventional-Commits](https://velog.io/@devshk447/%EB%82%98%EB%A7%8C-%EB%AA%B0%EB%9E%90%EB%8D%98-Conventional-Commits)

| 항목     | 설명                               |
| -------- | ---------------------------------- |
| feat     | 새로운 기능 추가 (Minor 버전 변경) |
| fix      | 버그 수정 (Patch 버전 변경)        |
| docs     | 문서 수정 (코드 변경 X)            |
| style    | 코드 포맷팅                        |
| refactor | 리팩토링 (동작 변화 X)             |
| test     | 테스트 추가, 수정                  |
| chore    | 기타 잡일 (빌드, 설정, 로그 등등)  |
| ci       | CI 설정 관련                       |
| build    | 빌드 시스템, 외부 의존성 변경      |
| perf     | 성능 개선                          |
| revert   | 기존 커밋 되돌림                   |

type(scope): 상세설명

**예시)**

-   feat(performance): 공연 생성 기능 추가

-   fix(performance): 공연 생성 기능 수정
